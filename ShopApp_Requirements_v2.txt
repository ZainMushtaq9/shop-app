SECTION 01 — PROJECT OVERVIEW
================================================================================

PURPOSE
-------
This app is a complete daily shop management system for small Pakistani
shopkeepers. It runs on Android, iOS, and in any web browser. The design
goal is that any shopkeeper — even with no education or tech experience —
can manage their entire business from their smartphone.

WHAT'S NEW IN THIS VERSION
---------------------------
Two major additions have been made based on real shopkeeper needs:

1. IN-APP GOOGLE SHEETS EDITOR
   Shopkeepers should NOT need to open Google Sheets on a laptop to make
   changes. They can edit, add, and delete products/prices/stock directly
   from inside the app using a built-in spreadsheet editor that looks and
   works like Google Sheets — but is part of the app itself.

2. COMPLETE LEDGER SYSTEM
   Every money transaction between the shopkeeper and buyers (customers)
   AND between the shopkeeper and suppliers is tracked as a proper
   debit/credit ledger — just like a traditional shop register (bahi khata)
   but digital. The app knows exactly:
   - Who owes money to the shopkeeper (buyers with udhaar)
   - Who the shopkeeper owes money to (suppliers on credit)
   - Complete running balance for every person

DESIGN PHILOSOPHY
-----------------
• SIMPLE       — Big icons, Urdu labels, minimal typing. Works for illiterate users.
• FREE         — 100% free tools only. No monthly fees.
• OFFLINE      — Works without internet. Syncs when connection available.
• URDU FIRST   — Full Urdu interface. All numbers, alerts, labels in Urdu.
• LEDGER-BASED — Every rupee is tracked. Nothing is lost.

SECTION 03 — IN-APP GOOGLE SHEETS MANAGER (NEW — MOBILE EDITING)
================================================================================

OVERVIEW
--------
The shopkeeper must NEVER need to open a laptop or a browser to edit their
Google Sheet data. All changes happen from inside the app. The app has a
dedicated "Sheets Manager" screen that shows data in a table/spreadsheet
format and allows full editing.

HOW IT WORKS (User Flow)
------------------------
Step 1: Shopkeeper opens app → taps "گوگل شیٹ" (Google Sheet) from menu.
Step 2: App shows a tabbed spreadsheet view inside the app:
        Tab 1: مصنوعات (Products)
        Tab 2: فروخت (Sales)
        Tab 3: گاہک (Customers)
        Tab 4: سپلائر (Suppliers)
Step 3: Shopkeeper can:
        • VIEW all rows in a scrollable table
        • TAP any cell to edit it directly
        • TAP "+ قطار شامل کریں" (Add Row) to add a new product
        • SWIPE LEFT on any row to delete it
        • TAP "محفوظ کریں" (Save) to push changes to Google Sheets
        • TAP "تازہ کریں" (Refresh) to pull latest data from Sheets
Step 4: All changes sync both to Google Sheets AND local SQLite database.

IN-APP SHEET EDITOR — DETAILED REQUIREMENTS
--------------------------------------------

A) PRODUCTS SHEET (مصنوعات)
Columns visible in app editor:
  Col 1: نام (Product Name) — text field
  Col 2: قیمت خرید (Purchase Price) — number field
  Col 3: قیمت فروخت (Sale Price) — number field
  Col 4: اسٹاک (Stock) — number field with +/- stepper
  Col 5: کم از کم اسٹاک (Min Stock Alert) — number field
  Col 6: زمرہ (Category) — dropdown selector

Actions available:
  • Add new product row → creates product in app AND Google Sheet
  • Edit any cell → tap cell → keyboard/number pad appears → auto-save
  • Delete row → swipe left → confirm → product archived in app + row deleted in Sheet
  • Sort by: Name, Price, Stock (tap column header)
  • Filter by: Category (dropdown filter at top)
  • Search bar to find product by name

B) SALES SHEET (فروخت) — READ ONLY in editor
  • Shopkeeper can VIEW all past sales in spreadsheet format
  • Cannot edit sales (to prevent fraud/errors)
  • Can export filtered date range to new Sheet

C) CUSTOMERS SHEET (گاہک)
Columns visible:
  Col 1: نام (Name) — text
  Col 2: فون (Phone) — number
  Col 3: باقی رقم (Balance) — auto-calculated, READ ONLY (shown in red if positive)
  Col 4: آخری خریداری (Last Purchase) — date, READ ONLY

Actions:
  • Add new customer row
  • Edit name and phone
  • Balance is auto-calculated from ledger — cannot be manually edited

D) SUPPLIERS SHEET (سپلائر)
Columns visible:
  Col 1: نام (Name) — text
  Col 2: فون (Phone) — number
  Col 3: واجب الادا (Amount Owed) — auto-calculated from ledger, READ ONLY
  Col 4: آخری خریداری (Last Purchase) — date, READ ONLY

SYNC BEHAVIOR
-------------
Direction 1: App → Google Sheets (PUSH)
  • Every save action in the in-app editor pushes to Google Sheets via API
  • Every new sale pushes a new row to Sales sheet
  • Every stock change updates the Products sheet stock column

Direction 2: Google Sheets → App (PULL)
  • When user taps "تازہ کریں" (Refresh), app reads all 4 sheets
  • New rows in Sheets = new records in app
  • Changed values in Sheets = updated records in app
  • Deleted rows in Sheets = archived records in app
  • Automatic background pull every 30 minutes when app is open

CONFLICT RESOLUTION
-------------------
If the same product was edited in the app AND in Google Sheets between syncs:
  • Show the shopkeeper a conflict card: "2 جگہ تبدیلی ہوئی" (Changed in 2 places)
  • Show both values side by side
  • Shopkeeper taps which value to keep
  • Resolved value is saved to both places

OFFLINE EDITING
---------------
  • If no internet, changes are saved locally with status "انتظار میں" (Pending)
  • A pending badge count shows on the Sheets menu icon
  • When internet returns, all pending changes are pushed automatically
  • Show "X تبدیلیاں اپ لوڈ ہوئیں" (X changes uploaded) notification

SECTION 04 — BUYER (CUSTOMER) LEDGER SYSTEM
================================================================================

CONCEPT — BAHI KHATA (بہی کھاتہ) DIGITAL
------------------------------------------
Every traditional shopkeeper keeps a physical register called "Bahi Khata"
where they record what each customer bought and what they paid. This app
replaces that register with a digital ledger that is more accurate, never
lost, and sends automatic reminders.

BUYER LEDGER — ALL TRANSACTION TYPES
--------------------------------------

TYPE 1: CREDIT SALE (ادھار / اُدھار فروخت)
  Meaning: Buyer takes goods but does NOT pay now. Money is owed TO shopkeeper.
  Effect: Buyer's balance INCREASES (goes into debit — they owe more)
  Entry in ledger: Date | "ادھار فروخت" | Items | Amount | DR | Running Balance

TYPE 2: CASH SALE (نقد فروخت)
  Meaning: Buyer takes goods and pays full amount in cash.
  Effect: No change to balance (zero net effect on ledger)
  Entry in ledger: Date | "نقد فروخت" | Items | Amount | PAID | Balance: 0

TYPE 3: PARTIAL PAYMENT AT SALE (جزوی ادائیگی)
  Meaning: Buyer pays some amount now, rest is ادھار.
  Effect: Balance increases by (Total - Amount Paid)
  Entry in ledger:
    Row 1: Date | "فروخت" | Items | Full Amount | DR
    Row 2: Date | "ادائیگی" | Cash Received | Amount Paid | CR
    Net Balance shown below

TYPE 4: PAYMENT RECEIVED (ادائیگی موصول)
  Meaning: Buyer comes and pays off some or all of their ادھار.
  Effect: Buyer's balance DECREASES (credit reduces the debt)
  Entry in ledger: Date | "ادائیگی موصول" | Amount Paid | CR | Running Balance

TYPE 5: RETURN / WAPSI (واپسی)
  Meaning: Buyer returns goods. Shopkeeper reduces the balance.
  Effect: Balance decreases
  Entry in ledger: Date | "مال واپسی" | Items | Amount | CR | Running Balance

TYPE 6: ADVANCE PAYMENT (پیشگی ادائیگی)
  Meaning: Buyer pays in advance for future purchases.
  Effect: Balance becomes negative (shopkeeper owes the buyer)
  Entry in ledger: Date | "پیشگی" | Amount | CR | Balance shown in GREEN

BUYER LEDGER — SCREEN DESIGN
------------------------------
Customer Detail Screen shows:

[HEADER CARD — RED if owed, GREEN if advance paid]
  Customer Name + Photo
  "کل باقی: Rs. 2,500" (Total Remaining: Rs. 2,500)
  [واٹس ایپ یاد دہانی] [ادائیگی لو] [بل دیکھو] buttons

[LEDGER TABLE — scrollable timeline]
  Date    | تفصیل          | ڈیبٹ (DR) | کریڈٹ (CR) | بیلنس
  --------|----------------|-----------|------------|-------
  03 Mar  | ادھار فروخت   | 1,500     |            | 1,500
  04 Mar  | ادائیگی موصول |           | 500        | 1,000
  05 Mar  | ادھار فروخت   | 800       |            | 1,800
  06 Mar  | ادائیگی موصول |           | 500        | 1,300
  07 Mar  | مال واپسی     |           | 200        | 1,100
  ...

[FOOTER]
  Total DR: Rs. 4,300  |  Total CR: Rs. 3,200  |  Balance: Rs. 1,100

ACTIONS IN BUYER LEDGER
------------------------
  • "+ ادائیگی لو" (Receive Payment) — record cash payment from buyer
  • "+ ادھار فروخت" (Credit Sale) — add a manual credit sale
  • "پی ڈی ایف بھیجو" (Send PDF) — generate ledger PDF and share on WhatsApp
  • "واٹس ایپ یاد دہانی" — pre-written Urdu message with balance
  • "فلٹر" — filter ledger by date range

PAYMENT REMINDER VIA WHATSAPP
-------------------------------
When shopkeeper taps "واٹس ایپ یاد دہانی", the app opens WhatsApp with
this pre-written message (in Urdu):

  "السلام علیکم [Customer Name] صاحب،
  آپ کے ذمے [Shop Name] کی طرف سے Rs. [Amount] باقی ہیں۔
  براہ کرم جلد ادائیگی فرمائیں۔ شکریہ۔"

BULK CUSTOMER SUMMARY
----------------------
All Customers screen shows:
  • Total amount owed BY all customers combined (big red number at top)
  • Total advance payments held (big green number)
  • Net receivable amount
  • List sorted by: Highest balance first (default)
  • Filter: Only show customers with pending balance

SECTION 05 — SUPPLIER LEDGER SYSTEM
================================================================================

CONCEPT
-------
When a shopkeeper buys goods from a supplier, they often do not pay the full
amount immediately. They take goods on credit and pay later. This section
tracks the complete financial relationship between shopkeeper and each supplier.

SUPPLIER LEDGER — ALL TRANSACTION TYPES
-----------------------------------------

TYPE 1: PURCHASE ON CREDIT (ادھار خریداری)
  Meaning: Shopkeeper takes goods from supplier, will pay later.
  Effect: Shopkeeper's payable balance INCREASES (shopkeeper owes supplier more)
  Entry: Date | "ادھار خریداری" | Items | Amount | DR (shopkeeper's debt)

TYPE 2: CASH PURCHASE (نقد خریداری)
  Meaning: Shopkeeper buys goods and pays full amount immediately.
  Effect: No change to outstanding balance
  Entry: Date | "نقد خریداری" | Items | Amount | PAID

TYPE 3: PAYMENT MADE TO SUPPLIER (سپلائر کو ادائیگی)
  Meaning: Shopkeeper pays off some or all debt to supplier.
  Effect: Shopkeeper's payable balance DECREASES
  Entry: Date | "ادائیگی کی" | Amount Paid | CR | Running Balance

TYPE 4: RETURN TO SUPPLIER (مال واپس سپلائر کو)
  Meaning: Shopkeeper returns damaged/wrong goods to supplier.
  Effect: Balance decreases (supplier owes the shopkeeper a credit)
  Entry: Date | "واپسی" | Items | Amount | CR | Running Balance

TYPE 5: ADVANCE PAYMENT TO SUPPLIER (پیشگی)
  Meaning: Shopkeeper pays supplier in advance.
  Effect: Balance becomes negative (supplier owes shopkeeper)
  Entry: Date | "پیشگی" | Amount | CR | Balance in GREEN

SUPPLIER LEDGER — SCREEN DESIGN
---------------------------------
Supplier Detail Screen shows:

[HEADER CARD — RED if shopkeeper owes, GREEN if supplier owes back]
  Supplier Name + Company
  "آپ کے ذمے: Rs. 8,000" (You Owe: Rs. 8,000)
  [ادائیگی کرو] [خریداری شامل کرو] [واٹس ایپ] buttons

[PURCHASE HISTORY + PAYMENT LEDGER — scrollable]
  Date    | تفصیل              | مال لیا (DR) | ادائیگی (CR) | بیلنس
  --------|--------------------|-------------|--------------|-------
  01 Mar  | ادھار خریداری     | 5,000       |              | 5,000
  02 Mar  | نقد خریداری       | 2,000       |              | 7,000  (paid on spot)
  03 Mar  | ادائیگی کی        |             | 3,000        | 4,000
  04 Mar  | ادھار خریداری     | 6,000       |              | 10,000
  05 Mar  | مال واپسی         |             | 2,000        | 8,000

[FOOTER]
  Total Purchased: Rs. 13,000  |  Total Paid: Rs. 5,000  |  Balance: Rs. 8,000

ACTIONS IN SUPPLIER LEDGER
----------------------------
  • "+ خریداری شامل کرو" (Add Purchase) — record new purchase from this supplier
    → Enter: items bought, total amount, payment method (cash/credit/partial)
    → If cash: balance unchanged. If credit: balance increases.
    → Stock auto-increases for items purchased
  • "+ ادائیگی کرو" (Make Payment) — record payment made to supplier
    → Enter: amount paid, payment method (cash/bank transfer)
    → Balance decreases accordingly
  • "واٹس ایپ" — send payment or purchase summary to supplier on WhatsApp
  • "رسید" (Receipt) — generate purchase receipt PDF

SUPPLIER PURCHASE ENTRY FLOW
------------------------------
When shopkeeper taps "+ خریداری شامل کرو":

Screen 1: SELECT SUPPLIER
  [Search bar] → tap supplier name → or tap "+ نیا سپلائر" (new supplier)

Screen 2: ADD ITEMS
  [Add Item button] → search product → enter quantity received → enter purchase price
  Multiple items can be added to one purchase entry
  [اگلا] (Next) button

Screen 3: PAYMENT DETAILS
  Total amount shown large
  Payment Method: [نقد] [ادھار] [جزوی]
  If partial: "ابھی کتنا دیا؟" (How much paid now?) — number field
  [محفوظ کرو] (Save)

Screen 4: CONFIRMATION
  Summary shown:
    Items received: X items
    Total value: Rs. X
    Paid now: Rs. X
    Remaining to pay: Rs. X (red if amount owed)
    Stock updated automatically ✓
  [واٹس ایپ سپلائر کو بھیجو] button

This is a new dedicated screen accessible from the main menu showing the

  List of all payments received today (from buyers)

  "30 دن سے زیادہ" (30+ days old debts) — highlighted in red
  Shows which customers have ادھار older than 30 days
  Shows which suppliers need to be paid

GOOGLE SHEETS SYNC FOR LEDGERS
--------------------------------

  Name | Phone | Total Purchased | Total Paid | Balance | Days Since Last Payment

  Name | Phone | Total Purchased | Total Paid | Balance | Last Purchase Date

SECTION 09 — GUI DESIGN GUIDE

DESIGN RULES (NON-NEGOTIABLE)
  1. ICON + LABEL: Every button shows icon AND text. Never icon-only.
  2. BIG TOUCH AREAS: Minimum 56x56dp touch target. 80dp for main actions.
  3. COLOR CODING:
       GREEN   = money received, profit, positive balance

SECTION 12 — GOOGLE DRIVE BACKUP
================================================================================

BACKUP FILE STRUCTURE
  Folder in Drive: "ShopApp Backup / [Shop Name]"
  File name: "backup_YYYY-MM-DD.json"
  File contents: Complete export of all SQLite tables as JSON

BACKUP SCHEDULE
  • Automatic: every night at 2:00 AM (workmanager background task)
  • Manual: Settings → "ابھی بیک اپ لو" button
  • Retention: Last 30 files kept, older deleted automatically

RESTORE PROCESS
  Settings → "بیک اپ سے بحال کرو" → Shows list of backup files →
  Tap file → Confirm → App restores all data → Requires app restart

WHAT IS BACKED UP
  • All products (including archived)
  • All customers and their full transaction ledger
  • All suppliers and their full transaction ledger
  • All sales and sale items
  • All expenses
  • App settings (shop name, currency, language, etc.)
  • All Google Sheets sync state
